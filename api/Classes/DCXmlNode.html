DCXmlNode class reference

DCDMNode : NSObject
DCXmlNode+AsString.m

Overview

DCXmlNode is the main class used to represent nodes in the Document Model (DM) tree.

The DCXmlParser and it's two delegates (DCXmlSubtreeParserDelegate and DCXmlDocumentParserDelegate) will generate the whole DM using instances of this class and the DCTextNode class. The bulk of the functionality of the api with regards to the document model is embedded in this class.

DCXmlNodes contain all the data required to represent a xml element in the object graph:

  • name

  • prefix

  • attributes

  • namespace declarations

  • sub elements which can be of any DMNode type.

DCXmlNode is also the main factory class of the DM, being about to create instances of itself as well as other nodes such as the DCTextNode.

Tasks

Properties

  • name property The name of the DCXmlNode as it would be printed in an xml stream.
  • prefix property Schema prefix.
  • defaultSchema property The defaultSchema is a schema specified without a prefix.

Constructors and factory methods

  • - initWithName: Creates a DCXmlNode with just a name.
  • - initWithName:prefix: Creates a DCXmlNode with a name and namespace prefix.
  • + createWithName: Factory method which creates a node (autorelease) with just a name and returns it.
  • + createWithName:prefix: Factory method which creates a node (autorelease) with a given name and prefix, then returns it.
  • + createWithName:value: Factory method which creates a node (autorelease) with a given name, and then sets a single DCTextNode under it containing the passed value.
  • + createWithName:prefix:value: Factory method which creates a node (autorelease) with a given name and prefix, and then sets a single DCTextNode under it containing the passed value.

Querying

  • - isEqualToName: Used during search loops to find child DCXmlNodes based on their names only.

Subnodes

  • - countNodes Returns the number of nodes within this node.
  • - xmlNodeWithName: Locates and returns a DCXmlNode based on it's name only.
  • - nodeAtIndex: Returns the DCDMNode at the specific index.
  • - addNode: Adds a new child DCDMNode to the end of the list of sub elements.
  • - addXmlNodeWithName: Adds a new child DCXmlNode to the end of the list of DCDMNodes.
  • - addXmlNodeWithName:prefix: Adds a new child DCXmlNode to the end of the list of DCDMNodes.
  • - addXmlNodeWithName:value: Adds a new child DCXmlNode to the end of the list of DCDMNodes.
  • - addXmlNodeWithName:prefix:value: Adds a new child DCXmlNode to the end of the list of DCDMNodes.
  • - hasXmlNodeWithName: returns true if the current DCXmlNode has a child DCXmlNode with the specified name.
  • - nodes Returns a NSContaining all the DCDMNodes.
  • - xmlNodesWithName: Searches the child nodes for DCXmlNodes with the specific name and returns a NSArray containing just those nodes.

Namespaces

  • - addNamespace:prefix: Adds a namespace and prefix to the current DCXmlNode.
  • - namespaces Returns an NSDictionary containing all the namespaces added to the current DCXmlNode.

Attributes

  • - setAttribute:value: Sets the name and value of an attribute.
  • - attributeValue: Returns the value for an attribute.
  • - attributes Returns a NSDictionary containing all the attributes.

Values

  • - value Shortcut method which returns the value of the first child DCTextNode or nil if there are no child DCTextNodes.
  • - addTextNodeWithValue: Adds an additional DCTextNode to the list of child nodes.
  • - setValue: Short cut method which clears all current child elements and then adds a single DCTextNode containing the passed text.

Xml output

  • - asXmlString Generates the DCXmlNode as a NSString.
  • - asPrettyXmlString Effectively the same as asXmlString: however this "pretty prints" it which is useful for logging purposes where readibility of the xml is the primary factor.

addNode:

Adds a new child DCDMNode to the end of the list of sub elements.

- (void)addNode:(DCDMNode *) element
Discussion

This is the main method for adding child nodes. If the passed node is a DCTextNode then it is just added to the nodesInOrder variable. if it's from the DCXmlNode hirachy then it is also added to the nodesByName dictionary, however this will only happen if there is not a node already present with the specified name. In other words, this method will only index the first occurance of a DCXmlNode with any give name.

addNamespace:prefix:

Adds a namespace and prefix to the current DCXmlNode.

- (void)addNamespace:(NSString *) aUrl prefix:(NSString *) aPrefix
Discussion

This will manifest as a namespace declaration in any produced xml. For example:

 DCXmlNode * node = [[DCXmlNode alloc] initWithName:@"abc"]; [node addNamespace:@"http://url.com" prefix:@"xyz"];
Will result in:
 <xyz:abc xlmns:xyz="http://url.com" />
